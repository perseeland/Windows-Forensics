ShellBags

Overview ▼

  • ShellBags record folder browsing history — Windows stores the size, position, and view settings of every folder a user opens in Explorer.
  • Forensically valuable because entries persist after folders are deleted — you can prove a user browsed a location even if it no longer exists.
  • Can reveal access to USB drives, network shares, deleted directories, and zip file contents.
  • Analysis must be done per user — each user's hives contain their own ShellBag data.
  • ShellBags are split across two hives: USRCLASS.DAT (primary — most complete) and NTUSER.DAT (secondary).

Registry Paths ▼

USRCLASS.DAT — Primary source
Local Settings\Software\Microsoft\Windows\Shell\Bags
  • Stores the actual bag data (window size, position, view settings) for each folder.
Local Settings\Software\Microsoft\Windows\Shell\BagMRU
  • Stores the folder paths as a tree structure — maps bag IDs to their corresponding folder locations.
  • This is the primary ShellBags location and contains the most complete browsing history.
NTUSER.DAT — Secondary source
HKCU\Software\Microsoft\Windows\Shell\BagMRU
  • Tracks folder browsing for Desktop and special shell locations.
HKCU\Software\Microsoft\Windows\Shell\Bags
  • Contains bag settings for Desktop-rooted paths and shell namespace locations.
Always load both USRCLASS.DAT and NTUSER.DAT together in ShellBag Explorer for a complete picture. USRCLASS.DAT contains the most extensive history.

RegRipper ▼

  • The shellbags plugin parses both Bags and BagMRU keys and outputs a list of browsed folder paths with timestamps.
  • Run separately against both USRCLASS.DAT and NTUSER.DAT for full coverage.
RegRipper
Run against USRCLASS.DAT (primary):
rip.exe -r C:\Cases\Analysis\Registry\UsrClass.dat -p shellbags > C:\Cases\Analysis\Registry\RR_Output\shellbags_usrclass.txt && type C:\Cases\Analysis\Registry\RR_Output\shellbags_usrclass.txt
Run against NTUSER.DAT (secondary):
rip.exe -r C:\Cases\Analysis\Registry\NTUSER.DAT -p shellbags > C:\Cases\Analysis\Registry\RR_Output\shellbags_ntuser.txt && type C:\Cases\Analysis\Registry\RR_Output\shellbags_ntuser.txt

ShellBag Explorer ▼

  • Open ShellBag Explorer and load the user's USRCLASS.DAT (File → Load Offline Hive).
  • The left panel shows the folder tree — browse exactly as the user did, including deleted and USB paths.
  • The right panel shows timestamps for each folder: first explored, last explored, and last interacted.
  • Use File → Export CSV to export all entries and open in Timeline Explorer for chronological sorting.
  • Sort by Last Write Time to quickly identify the most recently browsed locations during the attack window.
  • Look out for paths containing drive letters other than C: — these often indicate USB or external drive activity.
  • Network paths (starting with \\) indicate remote share access — note the IP or hostname for correlation.
ShellBags only record that a folder was browsed — not that specific files inside it were opened. Cross-reference with RecentDocs and LNK files for file-level evidence.
