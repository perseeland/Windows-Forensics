Scheduled Tasks

Overview ▼

  • Windows Scheduled Tasks allow programs and scripts to be executed automatically based on time, event triggers, or system conditions — making them a flexible and commonly abused persistence mechanism.
  • Tasks can be triggered by system events (login, startup, network connection), specific times (daily, weekly, on idle), or custom event log entries — giving attackers precise control over when their payload runs.
  • Scheduled tasks run under a configured user account or SYSTEM, and can be set to run whether or not the user is logged in.
  • Each task is stored as an XML file on disk AND has a corresponding entry in the registry TaskCache — both should be examined.
  • Deleted task XML files may still leave a registry stub in the TaskCache, and vice versa — cross-referencing both sources is important.

Locations ▼

File System — Task XML Files
C:\Windows\System32\Tasks\
  • Each scheduled task is stored as an XML file containing full configuration — triggers, actions, conditions, and the account it runs under.
  • The folder structure mirrors the Task Scheduler library tree — subfolders group tasks by category.
  • Use $MFT analysis to check the created timestamp of XML files — a newly created task file during the attack window is a strong indicator.
  • Deleted XML files may still be recoverable via the USN Journal or $MFT.
Registry — TaskCache
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tasks\
HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Schedule\TaskCache\Tree\
  • The Tasks subkey stores task metadata indexed by GUID — including the task path, last run time, and trigger information.
  • The Tree subkey mirrors the task folder structure and contains the task names.
  • A task entry in the TaskCache without a corresponding XML file indicates the XML was deleted — the registry stub may still reveal the task's path and last execution.

Key XML Elements ▼

<Actions>
  • Contains the command or script that executes — the most important element for forensic analysis.
  • Look for <Exec> elements with <Command> and <Arguments> values pointing to suspicious executables or encoded payloads.
<Triggers>
  • Defines when the task runs — at logon, on a schedule, at startup, on event, or on idle.
  • A trigger set to fire at every logon or on a repeating interval is a common attacker pattern.
<Principals>
  • Specifies the user account the task runs under and whether it requires the user to be logged in.
  • Tasks configured to run as SYSTEM or with highest privileges regardless of login status are high-priority flags.
<RegistrationInfo>
  • Contains the task author, description, and registration date.
  • Malicious tasks often have empty descriptions, generic author names, or dates that align with the attack window.

What To Look For ▼

  • Tasks with actions pointing to Temp, AppData, Downloads, or ProgramData — legitimate tasks rarely execute from these locations.
  • Tasks using LOLBins — e.g. powershell.exe -enc ..., cmd.exe /c ..., wscript.exe, mshta.exe, or certutil.exe.
  • Recently created task XML files — use $MFT to check the created timestamp against the attack window.
  • TaskCache entries without a corresponding XML file — the task was deleted but the registry stub remains.
  • Tasks in non-standard subfolders — legitimate Windows tasks are usually in \Microsoft\Windows\; tasks in the root or in custom folders are suspicious.
  • Tasks with no description or a generic name mimicking Windows system tasks.
  • Tasks set to run hidden with the highest privileges and regardless of user login status.

Tools ▼

1 Browse XML Files Directly
  • Navigate to C:\Windows\System32\Tasks\ in the mounted image and open suspicious XML files in a text editor.
  • Focus on the <Actions> and <Triggers> elements — these tell you what runs and when.
  • Cross-reference the task's XML file creation timestamp (from $MFT) with the attack timeline.
2 Registry Explorer
  • Load the SOFTWARE hive and navigate to the TaskCache keys.
  • Use the Bookmarks tab — Registry Explorer has pre-built bookmarks for Scheduled Tasks.
  • Check the last write time of TaskCache entries to identify recently created or modified tasks.
3 RegRipper
  • Use the taskcache plugin — this specifically targets the TaskCache\Tasks and TaskCache\Tree subkeys in the SOFTWARE hive, outputting task names, GUIDs, paths, and last run times.
  • The tasks plugin may also be available but targets a different or older task storage format — taskcache is the correct plugin for modern Windows systems (Vista and later).
RegRipper
Parse TaskCache (recommended):
rip.exe -r C:\Cases\Analysis\Registry\SOFTWARE -p taskcache > C:\Cases\Analysis\Registry\RR_Output\taskcache.txt && type C:\Cases\Analysis\Registry\RR_Output\taskcache.txt
