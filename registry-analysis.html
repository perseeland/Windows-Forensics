Windows Registry Overview

Hive Location ▼

  • \Windows\System32\config contains the hives for the target system.
  • Registry hives are binary files — they cannot be read as plain text and require a tool such as Registry Explorer or RegRipper to parse.
  • When analysing a mounted image, hives must be loaded offline — they are not live and will not reflect the current running state of a system.

System Specific Hives ▼

\Windows\System32\config\DEFAULT
  • Contains the default user profile settings applied before any user logs in.
  • Maps to HKEY_USERS\.DEFAULT.
\Windows\System32\config\SAM
  • Security Account Manager — stores local user account names and hashed passwords.
  • Maps to HKEY_LOCAL_MACHINE\SAM.
  • Key target for credential analysis.
\Windows\System32\config\SECURITY
  • Stores security policy settings and LSA secrets (cached credentials, service account passwords).
  • Maps to HKEY_LOCAL_MACHINE\SECURITY.
\Windows\System32\config\SOFTWARE
  • Contains installed software configuration, OS settings, and application data.
  • Maps to HKEY_LOCAL_MACHINE\SOFTWARE.
  • Rich source of installed program evidence and system configuration.
\Windows\System32\config\SYSTEM
  • Stores hardware configuration, device drivers, services, and startup settings.
  • Maps to HKEY_LOCAL_MACHINE\SYSTEM.
  • Key for identifying connected devices and installed services.

User Specific Hives ▼

\Users\<user>\AppData\Local\Microsoft\Windows\UsrClass.dat
  • Contains user-specific shell settings, COM class registrations, and file associations.
  • Also stores ShellBags — forensic artefacts that record folder browsing history, even for deleted folders.
  • Maps to HKEY_CURRENT_USER\Software\Classes.
\Users\<user>\NTUSER.DAT
  • The primary user hive — contains all user-specific settings.
  • Includes recently accessed files, typed URLs, search history, run history, and application preferences.
  • Maps to HKEY_CURRENT_USER.
  • One of the most valuable artefacts for user activity analysis.

Additional Info ▼

  • .log / .LOG1 / .LOG2 — Transaction logs that record changes to keys and value entries in the hive before they are committed.
  • These logs are used by Windows to recover a hive after a crash or unclean shutdown — the system replays the log to bring the hive to a consistent state.
  • Forensically important: the transaction logs may contain more recent changes that have not yet been written back to the main hive file. Always parse the hive together with its logs for a complete picture.
  • Tools such as Registry Explorer can automatically apply transaction logs when loading a hive.

Tools ▼

1 Registry Explorer
  • Load the hive files collected from the triage into Registry Explorer.
  • When loading, Registry Explorer detects .LOG1 / .LOG2 files and prompts you to apply them — always accept to get the most complete view.
  • A hive loaded without its transaction logs is a dirty hive and may be missing recent changes.
  • The #Values column shows the number of values in each key — useful for quickly spotting keys with data worth investigating.
  • The Bookmarks tab provides quick navigation to common forensic artefacts such as Run keys, recent files, and installed software.
2 RegRipper
  • A command-line tool that extracts and parses forensic artefacts from registry hives using plugins.
  • Each plugin targets a specific key or artefact — e.g. run keys, USB devices, recently accessed files.
  • Faster than manually browsing a hive when you know what you're looking for.
Run a single plugin against a hive:
rip.exe -r <hive> -p <plugin>
Run all plugins against a hive:
rip.exe -r <hive> -a
List all available plugins:
rip.exe -l
Output results to a file:
rip.exe -r <hive> -p <plugin> > output.txt
3 RegRipper Automation Script
List all files including hidden:
attrib *
Unhide a file:
attrib -h <file>
Clear system & hidden attributes from all files:
attrib -s -h *.*
Run all RegRipper plugins against every hive recursively:
for /r %i in (*) do (C:\Tools\RegRipper\rip.exe -r %i -a > %i.txt)
Open the output files in Notepad++ and use Find in All Open Documents (Ctrl+Shift+F) for a faster way to search across all results at once.
