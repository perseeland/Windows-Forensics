Evidence of Execution

BAM — Background Activity Moderator ▼

HKLM\SYSTEM\CurrentControlSet\Services\bam\State\UserSettings\{SID}
1 Overview
  • The Background Activity Moderator (BAM) is a Windows service introduced in Windows 10 version 1709 that controls and monitors background application activity.
  • From a forensic perspective, BAM records the full path and last execution time of every executable run on the system.
  • Each user has their own subkey identified by their SID — analysis must be performed per user.
  • Entries are stored in the SYSTEM hive — they survive user profile deletion and are not tied to the user's NTUSER.DAT.
  • BAM is one of the most reliable sources for proving program execution as it records the last run time directly in the registry.
BAM only records the last execution time — not the number of times a program was run. For execution counts use UserAssist or Shimcache.
2 What Can Be Identified
  • Full path of every executable that was run — including tools dropped in Temp, Downloads, or unusual locations.
  • Last execution timestamp for each program — useful for placing a tool within the attack timeframe.
  • Per-user execution history — identify which user account ran which program.
  • Evidence of living-off-the-land binaries (LOLBins) such as powershell.exe, cmd.exe, wscript.exe, or certutil.exe being used.
  • Tools or malware subsequently deleted — the BAM entry remains even after the file is removed from disk.
