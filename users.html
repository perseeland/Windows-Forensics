Users

Investigation Questions ▼

Which accounts were active during the attack timeframe?
Which account(s) were created?
Which accounts are Administrator group members?
Which users have profiles on the system? (Interactive logins)

SAM Hive ▼

C:\Windows\System32\config\SAM
1 Alias & Users Bookmarks
  • In Registry Explorer, navigate to the Bookmarks tab and find the Alias section — this shows all group names and their associated user SIDs, identifying which users belong to which group (e.g. Administrators).
  • Under the Users bookmark, export the data as a CSV and open it in Timeline Explorer for a cleaner and sortable view of all user accounts.
2 Account Timestamps
  • For each account you can view the last login time, account created date, and last password change.
  • Cross-reference these timestamps with your attack timeframe to identify suspicious account activity.

ProfileList — Identify Profiles on the System ▼

  • The ProfileList plugin identifies all user profiles created on the system from the SOFTWARE hive.
  • Useful for confirming which accounts have had interactive logins and have a profile directory on disk.
RegRipper
rip.exe -r C:\Cases\Analysis\Registry\SOFTWARE -p profilelist > C:\Cases\Analysis\Registry\RR_Output\profilelist.txt && type C:\Cases\Analysis\Registry\RR_Output\profilelist.txt
