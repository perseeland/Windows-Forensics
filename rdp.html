RDP — Remote Desktop Activity

Overview ▼

...TerminalServices-RemoteConnectionManager%4Operational.evtx
  • RCM — the inbound connection side. Event 1149 records the source user and source IP.
...TerminalServices-LocalSessionManager%4Operational.evtx
  • LSM — the session side. Logon, shell start, disconnect, reconnect, and logoff (21–25, 39, 40).
C:\Windows\System32\winevt\Logs\Security.evtx
  • Authoritative logon/logoff — 4624/4625 Type 10 (Type 7 on reconnect), plus 4634/4647/4778/4779.
C:\Windows\System32\winevt\Logs\System.evtx
  • Event 9009 (Desktop Window Manager exited) confirms a session was fully closed, not just disconnected.
  • RDP activity is spread across four logs — no single event tells the whole story, so the value is in correlating them on a timeline.
  • Logon Type 10 = RemoteInteractive (RDP). On a reconnect to an existing session it appears as Type 7 (unlock) instead.
  • Event 1149 is the earliest indicator of an inbound RDP attempt and carries the source IP — but it fires on the network connection, so it is not proof the credentials were valid. Pair it with a 4624 (success) or 4625 (failure) to know the outcome.
  • Disconnect ≠ logoff. A disconnect (24/4779) leaves the session running in the background; only a logoff (23/4634/4647) ends it. Attackers often disconnect rather than log off to keep their session alive.

Key Event IDs ▼

1149 — User Authentication Succeeded (RCM)
  • Earliest sign of an inbound RDP connection — records the source username, domain, and source IP address.
  • Despite the name, it fires on the network connection and does not confirm valid credentials — always pair with a 4624 / 4625 for the outcome.
4624 / 4625 — Logon Success / Failure (Security)
  • 4624 Type 10 = successful RemoteInteractive (RDP) logon; Type 7 on reconnect to an existing session.
  • 4625 Type 10 = failed RDP logon — repeated failures point to RDP brute-forcing.
21 / 22 / 25 — Logon / Shell / Reconnect (LSM)
  • 21 — session logon succeeded (records the source network address).
  • 22 — shell start notification received (the desktop finished loading — confirms an interactive session).
  • 25 — session reconnection succeeded.
23 / 24 / 39 / 40 — Logoff / Disconnect (LSM)
  • 23 — session logoff succeeded. 24 — session disconnected (session left running).
  • 39 — session X disconnected by session Y — indicates a purposeful disconnect (or one user kicking another).
  • 40 — session disconnected with a reason code; also correlates to reconnections.
4634 / 4647 / 4778 / 4779 — Session State (Security)
  • 4634 — an account was logged off. 4647 — user-initiated logoff.
  • 4778 — session reconnected to a window station (records source name + IP). 4779 — session disconnected from a window station.
9009 — Desktop Window Manager Exited (System)
  • Logged when the DWM exits — confirms a session was fully closed (logged off) rather than merely disconnected.

Tools ▼

1 Windows Event Log Explorer
  • Start in RemoteConnectionManager%4Operational.evtx and filter 1149 to list inbound connections and their source IPs.
  • Confirm the outcome in Security.evtx — 4624 Type 10 (success) or 4625 Type 10 (failure) at the matching time.
  • Build the session story in LocalSessionManager%4Operational.evtx — 21/22 (logon), 24/39/40 (disconnect), 25 (reconnect), 23 (logoff).
  • Sort each log by Date/Time and merge the events into one timeline keyed on the source IP and session.
