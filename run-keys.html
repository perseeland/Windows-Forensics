Run Keys

Overview ▼

  • Windows Run Keys are registry keys that specify programs to be executed automatically each time a user logs in or the system starts — one of the most commonly abused persistence mechanisms by attackers.
  • There are two variants: Run (executes every startup) and RunOnce (executes once on next startup, then the entry is deleted).
  • Entries can be set at the machine level (affects all users — HKLM) or the user level (affects only the specific user — HKCU).
  • Attackers commonly add entries pointing to malware in Temp, AppData, or unusual directories to maintain persistence across reboots.
  • Even if the malware file is deleted, the registry entry remains — this is forensic evidence of a persistence attempt.

Registry Locations ▼

HKLM — System-wide (all users)
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce
HKLM\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\Run
  • The WOW6432Node key is used by 32-bit applications running on a 64-bit system — do not overlook this as malware may target it specifically.
HKCU — Per-user (current user only)
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run
HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce
  • Found in the user's NTUSER.DAT hive — must be checked per user account.
  • Attackers often use HKCU entries as they do not require admin privileges to write.
Always check both HKLM and HKCU Run keys and for every user profile on the system — a single missed HKCU entry could be the persistence mechanism used by the attacker.

Registry Explorer ▼

  • Load the SOFTWARE hive (for HKLM entries) and each user's NTUSER.DAT hive (for HKCU entries).
  • Navigate to the Bookmarks tab — Registry Explorer has pre-built bookmarks for the Run and RunOnce keys.
  • Review each entry's value name (the label given to the entry) and value data (the full command or path that executes).
  • Check the last write time of the Run key — this records when an entry was last added or modified, helping place the persistence attempt within the attack timeline.
  • Flag entries pointing to paths in Temp, AppData, Downloads, or ProgramData — legitimate software rarely auto-starts from these locations.
  • Flag entries using cmd.exe, powershell.exe, wscript.exe, or mshta.exe to launch scripts — a common attacker technique.

RegRipper ▼

  • The run plugin parses all Run and RunOnce keys from both SOFTWARE and NTUSER.DAT hives.
  • Run against the SOFTWARE hive for system-wide HKLM entries, and against each user's NTUSER.DAT for per-user entries.
RegRipper
System-wide Run keys (HKLM):
rip.exe -r C:\Cases\Analysis\Registry\SOFTWARE -p run > C:\Cases\Analysis\Registry\RR_Output\run_hklm.txt && type C:\Cases\Analysis\Registry\RR_Output\run_hklm.txt
Per-user Run keys (HKCU):
rip.exe -r C:\Cases\Analysis\Registry\NTUSER.DAT -p run > C:\Cases\Analysis\Registry\RR_Output\run_hkcu.txt && type C:\Cases\Analysis\Registry\RR_Output\run_hkcu.txt
