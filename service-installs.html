Service Installs — System Event Log

Overview ▼

C:\Windows\System32\winevt\Logs\System.evtx
  • The Windows System event log records service installation, state changes, and failures — making it a key source for identifying malicious service-based persistence.
  • Event ID 7045 is the most forensically significant — it is written every time a new service is installed and records the full service configuration at the time of creation.
  • Critically, 7045 events are logged even if the registry service key is later deleted — giving you evidence of a malicious service even after an attacker has cleaned up.
  • Cross-reference 7045 events with the Services registry keys and $MFT timestamps to build a complete picture of when a service was installed and what it pointed to.

Key Event IDs ▼

7045 — New Service Installed
  • Logged every time a new service is installed on the system — the primary event for detecting malicious service installation.
  • Contains: Service Name, Service File Name (binary path), Service Type, Service Start Type, and Service Account.
  • Focus on the Service File Name — malicious services often point to paths in Temp, AppData, ProgramData, or use LOLBins.
  • The event timestamp tells you exactly when the service was installed — correlate with other activity in the attack timeline.
  • This event persists in the log even if the service and its registry key are subsequently deleted.
7036 — Service State Changed
  • Records when a service enters a running or stopped state.
  • Useful for identifying when a malicious service was first started and how frequently it ran — high-frequency restarts may indicate a watchdog mechanism.
7040 — Service Start Type Changed
  • Records when a service's start type was modified — e.g. changed from Manual to Automatic.
  • An attacker may change a legitimate service's start type to run their payload, or modify a malicious service to ensure it starts automatically.
7034 / 7031 — Service Crashed / Terminated Unexpectedly
  • Records when a service terminated unexpectedly or crashed.
  • Repeated crashes of an unknown service may indicate a malicious service that was killed by Defender or another process, or a poorly written payload.

Tools ▼

1 Windows Event Log Explorer
  • Open the System.evtx file and filter by Event ID 7045 to list all service installations.
  • Review the Service File Name field in each event — this is the binary path that was registered.
  • Sort by Date/Time and cross-reference with the known attack window.
2 EvtxECmd
  • Export the System log to CSV and open in Timeline Explorer — filter the EventId column for 7045 to isolate all service installs.
EvtxECmd
Parse System event log and export to CSV:
EvtxECmd.exe -f "C:\Cases\Analysis\C\Windows\System32\winevt\Logs\System.evtx" --csv "C:\Cases\Analysis\EventLogs" --csvf System.csv
