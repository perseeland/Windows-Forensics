RecentDocs

Overview ▼

NTUSER.DAT\Software\Microsoft\Windows\CurrentVersion\Explorer\RecentDocs
  • RecentDocs tracks recently opened files and folders accessed through Windows Explorer.
  • Analysis must be performed per user — each user's NTUSER.DAT contains their own RecentDocs entries.
  • Entries persist even after the files are deleted — the registry record remains, making it valuable forensic evidence.
  • The root key holds the most recently accessed files across all file types, ordered by the MRUListEx value.

What Can Be Identified ▼

  • File names of recently accessed documents, executables, archives, and other files — even if those files have since been deleted.
  • File types accessed — each extension has its own subkey (e.g. .pdf, .docx, .exe, .zip), allowing you to quickly filter by type.
  • Order of access — the MRUListEx value records the sequence in which files were opened, with the most recent at the top.
  • Access to removable media and network shares — file names may include drive letters or UNC paths revealing external storage usage.
  • Evidence of suspicious file activity — access to unusual file types (e.g. .exe, .ps1, .bat) or files in temp/appdata locations can indicate malicious behaviour.
  • Folder browsing — the Folder subkey tracks recently visited directories separately from files.
RecentDocs records file names only — not the full path. Use Shellbags or LNK files alongside RecentDocs for full path reconstruction.

Key Structure ▼

RecentDocs\ (root)
  • Contains the most recently accessed files across all types.
  • MRUListEx value defines the access order — index 0 is the most recent.
RecentDocs\.ext (e.g. .pdf, .exe, .docx)
  • Each file extension has its own subkey with its own MRU list.
  • Allows filtering by file type — useful for quickly finding all .exe or .zip files opened.
RecentDocs\Folder
  • Tracks recently visited folders and directories separately from files.
  • Can reveal browsing of sensitive directories or newly created folders.

RegRipper ▼

  • The recentdocs plugin parses all RecentDocs subkeys and outputs a list of recently accessed files grouped by extension.
  • Run against each user's NTUSER.DAT — repeat for every user account on the system.
RegRipper
rip.exe -r C:\Cases\Analysis\Registry\NTUSER.DAT -p recentdocs > C:\Cases\Analysis\Registry\RR_Output\recentdocs.txt && type C:\Cases\Analysis\Registry\RR_Output\recentdocs.txt
