ShimCache

Overview ▼

HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache\AppCompatCache
  • The ShimCache (also known as AppCompatCache) is a Windows mechanism used to track application compatibility — it records executable files that Windows has encountered on the system.
  • Stored in the SYSTEM hive and persists across reboots.
  • Entries are written to the registry on system shutdown — not in real time. If the system was powered off abruptly or crashed, the most recent entries may not have been committed.
  • ShimCache covers the entire system — it is not per-user like BAM.
  • The cache holds a limited number of entries (typically up to 1024 on Windows 10) — older entries are evicted as new ones are added.
On Windows 10, the presence of a ShimCache entry does not necessarily mean the file was executed — it may simply have been present on disk and accessed by the OS. On Windows 7 and earlier, an execution flag was recorded. Always corroborate with other execution artefacts.

What Can Be Identified ▼

  • Full path of executables that were present or run on the system.
  • Last modified time of the executable file — useful for detecting recently dropped or modified binaries.
  • Evidence of tools that have since been deleted from disk — the ShimCache entry remains even after the file is removed.
  • Presence of executables in unusual locations (Temp, Downloads, AppData) that may indicate malware or attacker tools.
  • Entries are ordered — the most recently encountered executables appear first, helping narrow down activity to a timeframe.

Tools ▼

  • AppCompatCacheParser by Eric Zimmermann is the recommended tool — exports results as a CSV for analysis in Timeline Explorer.
  • Each entry includes the file path, last modified time, and cache entry position (lower = more recently encountered).
  • The CSV output includes an Executed column — on Windows 7 and earlier this will show Yes or No confirming whether the file was run. On Windows 10 this column will show Unknown for most entries as the execution flag was removed from the cache format — presence alone cannot confirm execution.
  • Do not sort by the last modified timestamp when trying to determine execution order — the modified time reflects when the file was last changed on disk, not when it was run.
AppCompatCacheParser
Parse ShimCache and export to CSV:
AppCompatCacheParser.exe -f "C:\Cases\Analysis\Registry\SYSTEM" --csv "C:\Cases\Analysis\execution" --csvf ShimCache.csv
Registry Explorer
  • Load the SYSTEM hive into Registry Explorer.
  • Navigate to the Bookmarks tab and locate the AppCompatCache bookmark.
  • The entries will be displayed in order — use the cache entry position to determine sequence, not the timestamps.
