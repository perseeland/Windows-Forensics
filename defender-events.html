Windows Defender Event Log

Overview ▼

C:\Windows\System32\winevt\Logs\Microsoft-Windows-Windows Defender%4Operational.evtx
  • The Windows Defender Operational event log records all Defender activity — detections, actions taken, configuration changes, and protection status events.
  • Forensically valuable for understanding whether malware was detected during an attack, what action Defender took, and whether protection was disabled by an attacker to avoid detection.
  • A sudden sequence of protection disabled events followed by malware activity is a strong indicator of deliberate evasion.
  • Log entries include the threat name, file path, detection timestamp, and the action taken — quarantine, remove, or allow.

Key Event IDs ▼

1116 — Malware Detected
  • Defender detected a threat — records the threat name, severity, file path, and detection user.
  • First indicator that malicious software was present on the system — cross-reference the file path with $MFT and Amcache.
1117 — Action Taken
  • Records the action Defender took against a detected threat — Quarantine, Remove, Allow, or No Action.
  • If the action is Allow or No Action, the malware remained on the system — a critical finding.
1118 / 1119 — Remediation Failed / Succeeded
  • 1118 — Remediation attempt failed — the threat may still be present on the system.
  • 1119 — Remediation succeeded — the threat was successfully removed or quarantined.
1121 — Attack Surface Reduction Rule Triggered
  • An ASR rule blocked a suspicious behaviour — e.g. Office spawning child processes, credential theft from LSASS, or untrusted executables running.
  • ASR events can reveal attacker techniques even when the full payload was blocked.
5000 — Real-Time Protection Enabled
  • Real-time protection was turned on.
  • When seen immediately after a 5001 (disabled) event, it suggests the attacker re-enabled Defender after completing their activity to avoid leaving obvious traces of tampering.
  • A 5001 → malware activity → 5000 sequence in the timeline is a strong indicator of deliberate evasion and cleanup.
5001 — Real-Time Protection Disabled
  • Real-time protection was turned off — a critical event in any investigation.
  • If this event precedes malware activity, it strongly suggests the attacker deliberately disabled Defender before executing their payload.
5004 — Real-Time Protection Configuration Changed
  • A Defender configuration setting was modified — may indicate exclusions were added or scanning was reduced to allow malware to run undetected.
5007 — Configuration Changed
  • Records a change to Defender settings — includes both the old and new values.
  • Useful for identifying exactly what was changed and when — correlate with the attack timeline.
5010 / 5012 — Scanning Disabled
  • 5010 — Scanning for malware was disabled.
  • 5012 — Scanning for viruses was disabled.
  • Both are evasion indicators — flag these events and check what activity followed immediately after.

Tools ▼

1 Windows Event Log Explorer
  • Event Log Explorer is a GUI tool for browsing and analysing .evtx files offline — more powerful than the built-in Windows Event Viewer.
  • Open the Defender log directly: File → Open Log File → navigate to the collected .evtx file.
  • Use the Filter option to narrow down by Event ID — e.g. filter for 1116, 5001, 5007 to focus on detections and protection changes.
  • Use Find (Ctrl+F) to search for specific threat names, file paths, or keywords across all events.
  • Useful for quick interactive investigation before exporting to CSV for timeline analysis.
3 EvtxECmd
  • EvtxECmd by Eric Zimmermann parses .evtx log files and exports them as CSV for analysis in Timeline Explorer.
  • Use a maps file to translate raw event data into human-readable field names — EvtxECmd ships with maps for common Windows event logs including Defender.
  • In Timeline Explorer, filter by EventID to quickly isolate detection events (1116), protection disabled events (5001), or configuration changes (5007).
EvtxECmd
Parse Defender event log and export to CSV:
EvtxECmd.exe -f "C:\Cases\Analysis\C\Windows\System32\winevt\Logs\Microsoft-Windows-Windows Defender%4Operational.evtx" --csv "C:\Cases\Analysis\EventLogs" --csvf Defender.csv
