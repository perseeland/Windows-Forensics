Prefetch Analysis

Overview ▼

C:\Windows\Prefetch\*.pf
  • Windows Prefetch is a performance feature that caches data about recently executed programs to speed up future launches. Each executed application gets a .pf file named after the executable.
  • Unlike ShimCache, Prefetch files directly confirm execution — a .pf file will only exist if the program was actually run.
  • Prefetch files are named in the format: EXECUTABLE.EXE-XXXXXXXX.pf — the 8-character hex suffix is a hash of the full path from which the file was executed. Two entries for the same executable name but different hashes indicate it was run from different locations.
  • Prefetch is enabled by default on Windows workstations but is disabled by default on Windows Server.
  • The maximum number of Prefetch files is 128 on Windows XP/7 and 1024 on Windows 8+.
Check if Prefetch is enabled: HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\PrefetchParameters — EnablePrefetcher value: 0=Disabled, 1=App only, 2=Boot only, 3=Both enabled.

What Is Recorded ▼

Executable Name & Path Hash
  • The name of the executed program and a hash of the directory it was run from.
  • Multiple .pf files for the same executable name = ran from different locations.
Last Execution Time & Run Count
  • On Windows 8+, up to the last 8 execution timestamps are recorded per file.
  • Also records the total run count — how many times the program has been executed.
Referenced Files & Directories
  • A list of files and directories the executable accessed during the first 10 seconds of launch.
  • Can reveal DLLs loaded, config files read, and directories accessed — useful for understanding what a malicious tool did on first run.
Volume Information
  • The volume serial number and volume path from which the executable was run.
  • Can confirm execution from a USB drive or removable media if the volume serial doesn't match the system drive.

What Can Be Identified ▼

  • Confirmed execution — the existence of a .pf file proves the program was run, regardless of whether the original executable still exists on disk.
  • Execution timeline — up to 8 run times per executable on Windows 8+ let you map out when a tool was used throughout an attack.
  • Execution count — how many times a program was run; a high count for an unusual tool may indicate persistent or automated use.
  • Execution from removable media — the volume serial number reveals if a tool was run directly from a USB drive.
  • Deleted executables — the .pf file remains even after the original executable is deleted, proving it was present and executed.
  • Attacker tool behaviour — referenced files can reveal what the tool accessed, including staging directories, credential files, or lateral movement targets.

PECmd ▼

  • PECmd (Prefetch Explorer Command line) by Eric Zimmermann parses all .pf files and exports results as a CSV for analysis in Timeline Explorer.
  • Use the -d flag to process an entire directory of Prefetch files at once.
  • In Timeline Explorer, sort by LastRun and filter to the attack window to quickly identify tools that were executed.
  • Look for .pf files with unusual executable names, short run counts (ran once — may indicate a tool used briefly and deleted), or executions from non-standard paths.
  • PECmd also outputs a Referenced Files CSV — this lists every file and directory the executable accessed during its first 10 seconds of execution. This can reveal staging directories, credentials files accessed, lateral movement targets, and DLLs loaded — providing context on what the program did when it ran, not just that it ran.
  • PECmd generates a Timeline CSV (via the --csvf naming) that consolidates all execution timestamps across every .pf file into a single chronological view. Open this in Timeline Explorer to get a complete picture of all program executions in time order — especially useful for mapping out the sequence of attacker activity across the entire attack window.
PECmd
Parse all Prefetch files and export to CSV:
PECmd.exe -d "C:\Cases\Analysis\C\Windows\Prefetch" --csv "C:\Cases\Analysis\execution" --csvf Prefetch.csv
Parse a single Prefetch file:
PECmd.exe -f "C:\Cases\Analysis\C\Windows\Prefetch\MALWARE.EXE-XXXXXXXX.pf"
