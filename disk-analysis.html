$MFT

MFT — Master File Table ▼

$MFT — Root of the NTFS volume (e.g. C:\$MFT)
  • The Master File Table (MFT) is the core database of every NTFS volume — it contains an entry for every file and directory on the disk.
  • Each MFT entry is 1024 bytes and stores metadata about the file including timestamps, file size, permissions, and data location.
  • When a file is deleted, its MFT entry is marked as available but not immediately overwritten — the entry and its metadata often remain recoverable.
  • The MFT itself is a hidden system file located at the root of every NTFS volume.
  • Parsing the MFT gives a complete file system timeline — every file ever created, modified, accessed, or deleted.

Key MFT Attributes ▼

$STANDARD_INFORMATION
  • Contains the four NTFS timestamps: Created, Modified, MFT Modified, and Last Accessed.
  • These timestamps can be modified by user-mode applications — commonly manipulated in timestomping attacks.
  • Also stores file attributes (hidden, read-only, system) and owner SID.
$FILE_NAME
  • Also contains four NTFS timestamps — but these are updated by the kernel only and are much harder to tamper with.
  • Contains the file name and a reference to the parent directory MFT entry.
  • Comparing $STANDARD_INFORMATION vs $FILE_NAME timestamps is a key technique for detecting timestomping.
$DATA
  • Contains the actual file content or a pointer to where the data is stored on disk (data runs).
  • Small files may have their content stored directly within the MFT entry (resident data).
  • Larger files store data in clusters on disk with pointers recorded here (non-resident data).

Timestamps — MACB ▼

  • NTFS records four timestamps per file, commonly referred to as MACB.
  • MACB timestamps are taken from $STANDARD_INFORMATION — this is what forensic tools such as MFTECmd and Timeline Explorer display by default.
  • $FILE_NAME timestamps are used separately for timestomping detection only — they are not the primary MACB source.
M — Modified
  • When the file content was last changed.
A — Accessed
  • When the file was last read or opened. Often unreliable — Windows may disable last access updates for performance.
C — MFT Changed ($MFT Modified)
  • When the MFT entry itself was last changed (e.g. permissions, filename, attribute changes).
B — Born (Created)
  • When the file was first created on this volume. Note: copying a file sets a new Born time but preserves Modified time.
Timestomping — attackers modify $STANDARD_INFORMATION timestamps to hide activity. Always compare with $FILE_NAME timestamps — if they differ significantly, timestomping is suspected.
i Archive & Copy Timestamps — Not Always Timestomping
  • A Born timestamp newer than Modified does not always mean timestomping — this is a legitimate and common pattern caused by normal file operations.
Copying a file
  • The Born timestamp is reset to the time of the copy.
  • The Modified timestamp is preserved from the original.
  • Result: Born is newer than Modified — normal, not timestomping.
Extracting from an archive (.zip, .rar, .7z)
  • The Born timestamp is set to the extraction time.
  • The Modified timestamp may be preserved from inside the archive (tool-dependent).
  • Result: Born newer than Modified — expected and legitimate.
Moving a file (same volume)
  • All timestamps are generally preserved — Born and Modified remain unchanged.
Timestomping (suspicious)
  • $STANDARD_INFORMATION timestamps are backdated, but $FILE_NAME timestamps do not match — the key indicator.
  • Look for timestamps rounded to exact seconds — manual timestomping tools often produce this artefact.
  • A $SI Created timestamp that is earlier than the $FN Created timestamp is a strong indicator of manipulation.
Always check context — Born newer than Modified is suspicious on a system file but expected on a file extracted from a downloaded archive. Correlate with RecentDocs, browser history, and other artefacts before concluding timestomping.

MFTECmd ▼

  • MFTECmd by Eric Zimmermann parses the $MFT file and exports it as a CSV for analysis in Timeline Explorer.
  • Outputs all file entries including full paths, timestamps, file sizes, and resident data.
  • Open the output CSV in Timeline Explorer and filter/sort by timestamp to build a file system activity timeline.
  • Look for files created or modified during the attack window and files in suspicious locations (Temp, AppData, Downloads).
  • MFTECmd includes a SI<FN column in the CSV output — a value of True means the $STANDARD_INFORMATION timestamps are earlier than the $FILE_NAME timestamps, which is a strong indicator of timestomping. Filter this column in Timeline Explorer to quickly identify suspicious entries.
MFTECmd
Parse $MFT and export to CSV:
MFTECmd.exe -f "C:\Cases\C\$MFT" --csv "C:\Cases\Analysis\NTFS" --csvf MFT.csv
Dump full details for a specific MFT entry (--de flag):
MFTECmd.exe -f "C:\Cases\C\$MFT" --de 624
i The --de Flag
  • --de dumps the full details of a specific MFT entry by its entry number or entry-sequence number.
  • Format is Entry or Entry-Seq as decimal or hex — e.g. 5, 624-5, or 0x270-0x5.
  • Useful when you have identified a file of interest in the CSV output and want to inspect its full MFT record in detail — all attributes, timestamps, data runs, and flags.
  • The entry number can be found in the EntryNumber column of the MFTECmd CSV output.
Deleted files are included in the standard CSV output — look for the InUse column set to False to identify entries that have been deleted.
