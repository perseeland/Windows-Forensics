Sysmon — Operational Log

Overview ▼

C:\Windows\System32\winevt\Logs\Microsoft-Windows-Sysmon%4Operational.evtx
  • Sysmon (System Monitor) is a Sysinternals tool that must be installed beforehand — when present it provides far richer, more reliable telemetry than native Windows logging.
  • It records process creation with hashes, network connections, file writes, registry changes, and DNS queries — the core building blocks of an attack timeline.
  • What gets logged is config-driven (the XML config, e.g. SwiftOnSecurity / Olaf Hartong) — absence of an event may just mean it wasn't configured, not that it didn't happen.
  • Every event carries a ProcessGuid that uniquely identifies a process across reboots — use it to pivot between a process's creation (1) and everything it did afterwards (3, 11, 22).

Key Event IDs ▼

1 — Process Creation
  • The most valuable Sysmon event. Logs every process start with the full command line, Image path, working directory, and the executing user.
  • Records the ParentImage and ParentCommandLine — letting you spot suspicious chains like winword.exe → powershell.exe → cmd.exe.
  • Includes file Hashes (MD5/SHA256/IMPHASH) for the image — pivot straight to VirusTotal or a known-bad list.
The ProcessGuid field here is the key pivot — copy it and filter the log on that value to surface every network connection, file write, and DNS query that this same process generated.
3 — Network Connection
  • Logs TCP/UDP connections, recording SourceIp, DestinationIp, DestinationPort, DestinationHostname, and the originating Image.
  • Key for spotting C2 / beaconing — regular connections to an unusual external IP, or a connection made by a process that has no business talking to the network (e.g. powershell.exe, rundll32.exe).
11 — File Create
  • Logs when a file is created or overwritten — records the creating Image, the TargetFilename, and the CreationUtcTime.
  • Catches dropped payloads (in Temp/AppData/ProgramData), tool transfers, and ransomware note/encrypted-file creation.
12 — Registry Object Create / Delete
  • Logs registry key creation and deletion — the EventType field shows CreateKey or DeleteKey, with the TargetObject (key path) and originating Image.
  • Useful for spotting new persistence keys being created, or an attacker deleting keys to clean up.
13 — Registry Value Set
  • Logs registry value modifications — records the TargetObject (full value path) and the Details (the data written).
  • The go-to event for registry persistence — watch Run/RunOnce keys, service ImagePaths, and other autostart locations being written.
22 — DNS Query
  • Logs DNS lookups, recording the QueryName, the QueryResults (resolved addresses), and the Image that made the request.
  • Surfaces C2 domains, DGA traffic, and exfil endpoints — and ties the resolved domain directly to the process responsible.

Tools ▼

1 Windows Event Log Explorer
  • Open Microsoft-Windows-Sysmon%4Operational.evtx and filter by Event ID — start with 1 (process creation) to anchor the timeline.
  • Pull the ProcessGuid from a suspicious process 1 event, then filter on it to gather its 3, 11, 22 events into a single session view.
  • Sort by Date/Time and correlate with the known attack window.
