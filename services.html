Windows Services

Overview ▼

HKLM\SYSTEM\CurrentControlSet\Services\
  • Windows Services are long-running background processes that can start automatically at boot — making them a powerful persistence mechanism for attackers.
  • All service configurations are stored under HKLM\SYSTEM\CurrentControlSet\Services\ — each service has its own subkey.
  • Malicious services often run as SYSTEM — the highest privilege level on Windows — making them difficult to detect and stop.
  • Unlike Run Keys, services start before user login — they persist even if no user ever logs in.
  • The last write time of a service registry key records when it was created or last modified — useful for placing the installation of a malicious service within the attack timeline.

Key Registry Values ▼

ImagePath
  • The full path to the service executable — the most important value for forensic analysis.
  • Malicious services often point to executables in Temp, AppData, ProgramData, or unusual system directories.
  • May also use command-line arguments to launch scripts via cmd.exe, powershell.exe, or wscript.exe.
Start
  • Controls when the service starts — key values to note:
  • 0 = Boot — loads by the bootloader before the kernel
  • 1 = System — loaded by the kernel during boot
  • 2 = Automatic — starts automatically at login (most common for malicious services)
  • 3 = Manual — only starts when explicitly called
  • 4 = Disabled
ObjectName
  • The account the service runs as — typically LocalSystem, LocalService, or NetworkService.
  • A service running as LocalSystem has the highest privileges — flag any unknown service using this account.
  • Services using a custom domain or local account may indicate lateral movement or credential abuse.
DisplayName & Description
  • Attackers often use names that mimic legitimate Windows services — e.g. slightly misspelled names or generic descriptions like "Windows Update Helper".
  • An empty or suspicious description is a red flag for a malicious service.

What To Look For ▼

  • ImagePath pointing to non-standard locations — Temp, Downloads, AppData, ProgramData, or the root of C:\ are suspicious.
  • Services with Start = 2 (Automatic) that were recently created — cross-reference the key's last write time with the attack window.
  • Services running as LocalSystem with an unknown or suspicious name.
  • ImagePath using LOLBins — e.g. cmd.exe /c powershell.exe -enc ... or wscript.exe C:\Temp\malware.vbs.
  • Missing service binary — if the ImagePath points to a file that no longer exists on disk, it may have been deleted after use or the service may be a remnant of a cleanup attempt.
  • Misspelled or lookalike service names — e.g. svchost32 instead of svchost, or extra spaces in the name.

Tools ▼

1 Registry Explorer
  • Load the SYSTEM hive and navigate to CurrentControlSet\Services\.
  • Use the Bookmarks tab — Registry Explorer has a pre-built bookmark for Services.
  • Check the last write time of each service key — recently created or modified services during the attack window are priority targets.
  • Export to CSV for a sortable view of all services and their ImagePath values.
2 RegRipper
  • The services plugin outputs all service configurations including ImagePath, Start type, and ObjectName.
RegRipper
rip.exe -r C:\Cases\Analysis\Registry\SYSTEM -p services > C:\Cases\Analysis\Registry\RR_Output\services.txt && type C:\Cases\Analysis\Registry\RR_Output\services.txt
