PowerShell — Operational & Classic Logs

Overview ▼

C:\Windows\System32\winevt\Logs\Microsoft-Windows-PowerShell%4Operational.evtx
  • Modern PowerShell log — home of Script Block Logging (4104) and Module Logging (4103). The richest source of what was actually executed.
C:\Windows\System32\winevt\Logs\Windows PowerShell.evtx
  • Classic/legacy log — engine lifecycle (400/403) and pipeline details (800). Always present, even when modern logging is off.
  • PowerShell is the attacker's tool of choice — used for downloads (IEX (New-Object Net.WebClient).DownloadString), in-memory execution, recon, and post-exploitation frameworks (Empire, Cobalt Strike, PowerSploit).
  • Script Block Logging (4104) is the single most valuable artefact — it records the full script content after deobfuscation, so even Base64/obfuscated payloads appear in cleartext.
  • Crucially, even when logging is not fully enabled, PowerShell auto-logs script blocks it deems suspicious as 4104 events at Warning (level 3) — so you often have evidence anyway.
  • Cross-reference PowerShell events with 4688 (process creation) and network/file artefacts to reconstruct the full command sequence.

Key Event IDs ▼

4104 — Script Block Logging (Operational)
  • The most important PowerShell event. Records the content of each script block as it is compiled and executed.
  • PowerShell logs the block after it has been deobfuscated/decoded — so -EncodedCommand Base64, string concatenation, and other obfuscation are captured in cleartext.
  • Filter on Level = Warning (3) first — these are blocks PowerShell flagged as suspicious and are logged even without full logging configured.
  • Long scripts are split across multiple 4104 events — reassemble using MessageNumber of MessageTotal and the shared ScriptBlockId.
Enabled via: HKLM\SOFTWARE\Policies\Microsoft\Windows\PowerShell\ScriptBlockLogging\EnableScriptBlockLogging = 1. Worth checking — if it was recently disabled, that itself is an IOC.
4103 — Module / Pipeline Logging (Operational)
  • Records pipeline execution details — the commands invoked and their bound parameter values, plus the user and host.
  • Complements 4104: where script block logging shows the code, module logging shows the command + arguments that ran through the pipeline.
400 / 403 — Engine Lifecycle (Classic)
  • 400 — a PowerShell engine started (a session began). 403 — the engine stopped.
  • The 400 event records the HostApplication (full launching command line) and the EngineVersion.
40961 / 40962 — Console Host Startup (Operational)
  • 40961 — the PowerShell console host is starting; 40962 — the console is ready for input.
  • Useful as a lightweight marker of when PowerShell was launched and by which user, even before any commands run.

Tools ▼

1 Windows Event Log Explorer
  • Open Microsoft-Windows-PowerShell%4Operational.evtx and filter by Event ID 4104 — then narrow to Level = Warning for the auto-flagged suspicious blocks.
  • Open Windows PowerShell.evtx and filter 400 — check the EngineVersion field for downgrade attempts.
